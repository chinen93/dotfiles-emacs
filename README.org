* Dotfiles-Emacs

A personal Emacs configuration that uses [[https://orgmode.org/][org-mode]] babel tangling]] to generate the final =init.el= file. Each configuration module is written as an =.org= file containing embedded Emacs Lisp code blocks that get tangled into separate =.el= files.

** Why Org-Mode?

Using org-mode for configuration provides:
- Literate programming: documentation and code together
- Easy organization: hierarchical structure with properties
- Tangling: code blocks automatically become valid Emacs Lisp
- Export flexibility: can export to multiple formats

* Features

| Category | Description |
|----------|-------------|
| **Keybindings** | Custom prefix key =C-v= or =M-v= (avoids =C-x= pinky strain) |
| **Editing** | Evil mode for Vim keybindings integration |
| **Completion** | Ivy + Counsel for flexible completion |
| **Org-mode** | Extensive personal organization system |
| **Hydra** | Modular keybinding menus |
| **Snippets** | Custom Yasnippet snippets for multiple modes |

* Installation

** Linux/macOS

#+BEGIN_SRC sh
  git clone https://github.com/chinen93/dotfiles-emacs.git
  cd dotfiles-emacs
  ./bin/install.sh
#+END_SRC

The install script:
1. Creates symlinks from =~/emacsConfig/= and =~/emacsSnippets/=
2. Copies =init-emacs.org= to =~/.emacs.d/=

** Windows

#+BEGIN_SRC sh
  cd bin/windows
  install_windows.bat
#+END_SRC

Copies configuration files to =%USERPROFILE%\.emacs.d\=

** Manual Tangling (Optional)

#+BEGIN_SRC sh
  ./bin/init_emacs_tangle.sh
#+END_SRC

Re-tangles all =.org= configuration files into =.el= files.

* Configuration Load Order

The =init-emacs.org= loads modules in this sequence:

1. =init-setup-package.org= — Package manager setup
2. =init-basicConfig.org= — Base paths, global variables
3. =init-keyBinding.org= — Custom keymap
4. =init-coding.org= — Programming modes
5. =init-completion.org= — Completion framework
6. =init-customFunctions.org= — Helper functions
7. =init-hydra.org= — Hydra menus
8. =init-orgmode.org= — Org-mode configuration
9. =init-evil.org= — Evil mode (Vim keybindings)
10. Plus 15+ additional modules

* Custom Keybinding System

Instead of using =C-x= as the prefix command (which can cause repetitive strain), this config provides =C-v= or =M-v= as an alternative prefix:

#+BEGIN_SRC emacs-lisp
  (define-prefix-command 'my-prefix-command)
  (define-key my/keymap (kbd "C-v") 'my-prefix-command)
  (define-key my/keymap (kbd "M-v") 'my-prefix-command)
#+END_SRC

All custom bindings live in =my/keymap= and are exposed via =my-keys-minor-mode=.
